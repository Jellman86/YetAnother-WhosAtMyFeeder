# Pre-3.0 Shortlist Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement the pre-3.0 shortlist: OAuth token encryption at rest, OAuth cleanup on logout, a typed `/api/settings` response model, and a 365-day server-side date range cap.

**Architecture:** Keep the current OAuth storage schema and add app-layer token encryption/decryption helpers that transparently upgrade legacy plaintext rows on read. Add a concrete `SettingsResponse` model matching the current flat payload shape, then enforce a shared date-range guard in the explicit date-range species/stats endpoints.

**Tech Stack:** FastAPI, Pydantic, aiosqlite, Alembic-compatible SQLite schema, pytest.
